## Version History

### Version Format

P31 uses semantic versioning: `MAJOR.MINOR.PATCH`

- **MAJOR**: Breaking changes
- **MINOR**: New features (backward compatible)
- **PATCH**: Bug fixes (backward compatible)

### Component Versions

Components may have independent versioning:
- P31 Tandem: `1.0.0`
- P31 Spectrum: `1.0.0`
- P31 Buffer: `1.0.0`
- P31 NodeZero: `1.0.0`

## Release Process

1. Update version numbers
2. Update CHANGELOG.md
3. Create git tag
4. Build and test
5. Launch
