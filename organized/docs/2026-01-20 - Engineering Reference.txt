The "Phenix Navigator" has evolved from a theoretical prototype into a rigorous forensic instrument. Following the critical audit of the ATECC608A (vulnerable to laser fault injection) and the regulatory constraints of the EU (915MHz illegality), we have executed a Hard Fork of the architecture.
* The Hardware: Replaced ATECC608A with NXP SE050 (EAL6+). Implemented software-switchable 868/915 MHz LoRa.
* The Software: Adopted the Xiaozhi v2.1.0 codebase as the foundation, leveraging its Model Context Protocol (MCP) and Layered UI (Sprites) to visualize the "Cognitive Shield."
* The Logic: Integrated the SIC-POVM geometry as a local MCP Server, ensuring quantum-grade entropy without cloud dependence.
2. Hardware Specifications (Tier 1 Verified)
A. The Core (Empirical Constraints)
* MCU: ESP32-S3R8 (Dual Core, 16MB Flash, 8MB Octal PSRAM).
   * Critical: Must use qio_opi mode. Legacy qio_qio builds will crash the display.
* Power Management (PMIC): AXP2101 (I2C 0x34).
   * Controls: ALDO1 (Screen BL), ALDO2 (LoRa VCC), ALDO4 (AVDD).
* Secure Element (The Vault): NXP SE050 (I2C 0x48).
   * Replacement: Replaces ATECC608A. Requires sss-edgelock middleware.
   * Function: ECDSA Signatures (P-256), True RNG, SCP03 Encrypted Bus.

	C. Signal Integrity Fixes ("Phantom" Protocol)
1. Display: Enable CONFIG_LCD_RGB_BOUNCE_BUFFER_MODE=1. This forces the driver to copy PSRAM data to internal SRAM before DMA transfer, preventing artifacts during Wi-Fi bursts.
2. Audio: Implement Soft_Mute() on the ES8311. Ramp volume to 0 before stopping the I2S clock to eliminate "pops."
4. Operational Strategy: The "Genesis Gate"
A. The "Catcher's Mitt" (Input Sanitization)
* Mechanism: A 16KB RingBuffer in Internal SRAM.
* Logic: All LoRa/BLE packets are dumped here raw.
* Task: Sanitizer_Task (Low Priority) parses JSON schema. Malformed or hostile packets (high-entropy bursts) are dropped before reaching the MCP layer.
B. The "Voltage Strip" (UI Theme)
* Visual: A transparent Sprite overlay (Layer 2) on top of the main UI.
* Logic:
   * Blue: System Nominal. Coherence High.
   * Red: Threat Detected. (Bad Signature / High RSSI Jitter).
   * Purple: Totem Sync Active (Bluetooth Handshake).
C. Regulatory Compliance (EU/US)
* Firmware: Includes a compile-time flag #define REGION_EU868.
* Logic: If defined, sets SX1262 frequency to 868.1 MHz and enforces <1% duty cycle (LBT/AFA) to comply with ETSI EN 300 220.
5. Security & Governance
A. The "Vault" (NXP SE050)
* Identity: The device's private key (P-256) is generated inside the SE050 during the "Genesis" ritual. It never leaves the chip.
* Attestation: The SE050 signs a "Health Report" (Voltage, CRC errors, Uptime). The mesh accepts this signature as proof of node integrity.
B. Abdication (No Backdoors)
* JTAG: The DIS_JTAG eFuse is burned.
* Secure Boot: Enabled (RSA-3072).
* Admin Keys: The signing key is destroyed after the "Golden Image" is flashed. The firmware is immutable (except via signed OTA).
