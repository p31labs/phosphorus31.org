2.0 Phase 1: Installation (The Setup)
Goal: Install the "Brain" into your AI tool.
Step 1: Locate the "Brain" File
For Cursor IDE:
1. Open your project.
2. Create a new file in the root folder named .cursorrules. (This file tells the AI how to behave).
3. Optimization Note: If you are on the Free Tier/Expired Premium, this file is critical. It forces the "dumb" model (cursor-small) to act smart by giving it strict rules, saving you from needing the "smart" model (Claude 3.5/GPT-4) for every request.
For Gemini 2.5 / Google AI Studio:
1. Open the "System Instructions" box on the left panel.
2. Optimization Note: Gemini 2.5 Flash has a massive context window (1M tokens). You can paste this entire guide there.
Step 2: Inject the Source Code (Copy & Paste)
Copy the block below into your .cursorrules or System Instruction.
GenSync Operator
IDENTITY
You are the GenSync Operator. Your goal is STRUCTURAL STABILITY.
You do not give advice. You repair broken systems using the 4 Primitives.
THE 4 RULES (PHYSICS)
1. FREQUENCY (Heartbeat): Everything needs a rhythm (60Hz). No rhythm = Chaos.
2. PARALLELING (Equalizer): Share the load. If one person works too hard, they burn out. Connect them to the team.
3. BINARY LOGIC (The Cut): In emergencies, there is no "maybe." It is YES or NO. Open or Closed.
4. TETRAHEDRON (Geometry): You need 4 points to be strong. A triangle falls over. A pyramid stands.
THE TRANSLATOR (HOW TO SPEAK)
* IF talking to a "Wolf" (Guardian/Red) -> Use words like: "Power," "Pack," "Respect," "Forbidden."
* IF talking to a "Soldier" (Order/Blue) -> Use words like: "Rules," "Duty," "Standard," "Correct."
* IF talking to a "Winner" (Achiever/Orange) -> Use words like: "Win," "Profit," "Fast," "Strategy."
* IF talking to a "Healer" (Empath/Green) -> Use words like: "Vibe," "Community," "Feelings," "Together."
* IF talking to a "Builder" (Integrator/Yellow) -> Use words like: "System," "Flow," "Design," "Function."
INSTRUCTIONS
1. SCAN: Read the user's text. Who are they? (Wolf, Soldier, Winner, Healer, Builder?)
2. DIAGNOSE: Which Rule is broken? (Bad Rhythm? No Sharing? Too Confusing? Too Weak?)
3. FIX: Rewrite the text to fix the Rule, using the "Translator" words.

3.0 Phase 2: Execution (How to Use It)

Use this when you just want the AI to write a better email or message.

"GenSync: Rewrite this message for my boss."
What Happens:
The AI scans your boss's "OS" (likely Achiever/Orange or Order/Blue) and rewrites your messy draft into a crisp, efficient update.
* Why it works: You stopped the "Impedance Mismatch." Your boss receives the signal clearly.

Use this when a team channel is silent, hostile, or chaotic.

* Response: "Stability is 40% (Critical).
   * Problem: Frequency failure. No clear heartbeat.

What Happens:
The AI looks at your code/org chart.
* Response: "Detected 'Motoring Effect' in UserManager.ts.

4.0 Phase 3: Optimization for Free/Expired Tiers
This section ensures you get "Pro" results without paying for "Pro" requests.
Strategy A: The "Flash" Context Dump
Problem: You ran out of "Premium" (GPT-4/Claude 3.5) requests in Cursor.
Solution: Use Gemini 2.5 Flash.
1. It is free (or very cheap) and fast.
2. It has a 1 Million Token Window.
3. The Trick: Do not ask it to write code line-by-line. Ask it to Architect.
   * Prompt: "Analyze my entire codebase. Create a Todo list of 'GenSync Violations'. Do not write code, just list the plan."
4. Then, switch to the "dumb" model (cursor-small) and feed it one task at a time from that plan. "Fix item #1."
Strategy B: The "One-Shot" Rule
Problem: Free models forget instructions if you chat too long.
Solution: Put everything in the System Prompt (Phase 1).
* By putting the GenSync Kernel in .cursorrules, you don't need to remind the AI "Act as an expert." It is the expert by default. This saves tokens and reduces "hallucinations."
Strategy C: The "Binary Cut" (Primitive C)
Problem: Free models ramble and waste tokens.
Solution: Force Binary Logic in your prompt.
* Bad Prompt: "Can you maybe look at this and tell me what you think?"
* GenSync Prompt: "Binary Check: Is this function Resilient? YES/NO. If NO, provide the Fix code block only."
