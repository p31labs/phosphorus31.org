8.1 The "Catcher's Mitt" Logic (Redis Streams)
Standard Pub/Sub messaging is insufficient for neurodivergent users because it is "fire-and-forget." If the user is overwhelmed and goes "offline" (Airplane mode), messages are lost, creating FOMO and anxiety. The architecture requires Redis Streams, which provides an append-only log that persists data until the user is ready to process it.1
The Looper Pattern (Python/FastAPI):
The system uses a middleware "Looper" to manage message flow:
   1. Ingestion: Incoming message $\rightarrow$ Encrypted (AES-256) $\rightarrow$ xadd to Redis Stream stream:{user_id}.
   2. The Delay (60s): The worker monitors the stream. It waits 60 seconds from the last message to ensure the "thought" is complete (The Catcher's Mitt). This prevents the "Machine Gun Effect" of rapid-fire notifications.
   3. Batching: It concatenates fragmented messages into a single context.
   4. Processing: It sends the batch to the Cognitive Shield Ollama model.
   5. Output: It returns the structured JSON with Urgency Score and Cognitive Cost.
8.2 Security & Encryption
Even locally, the protocol demands a "Privacy by Design" approach to protect the user's psychological safety.1
