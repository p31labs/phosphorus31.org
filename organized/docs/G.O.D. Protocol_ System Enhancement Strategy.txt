2.1 The "Whale Song" Transport Protocol (WSTP)
The "Doing More With Less" coding style 1 and the constraints of LoRa bandwidth (0.350 kbps at SF12) necessitate a rejection of standard JSON payloads for inter-device communication. JSON is verbose, string-heavy, and requires expensive parsing cycles that drain battery and flood the airwaves. Instead, we define a strict Binary Schema utilizing Protocol Buffers (Protobuf) optimized for embedded systems, specifically leveraging the nanopb library for C/C++.3
2.1.1 Hybrid Serialization Strategy
We propose a hybrid serialization architecture to balance the conflicting needs of the "Iron Core" (firmware) and the "Fluid Interface" (software):
1. Inter-Device (LoRa/Mesh): Custom Bit-Packed Structs or nanopb with strict int_size options to minimize over-the-air (OTA) byte count.4
2. Intra-Device (Shield Internal): Type-Safe JSON (for React state persistence) backed by EncryptedBlob branded types to ensure no plaintext leakage.5
2.1.2 Optimization for LoRa (The "Whale Song")
Standard Protobuf implementations, while compact, can still exceed the stringent 915 MHz "Whale Song" limits when metadata overhead is considered. A 50-byte packet at Spreading Factor 12 (SF12) consumes approximately 3 seconds of airtime.6 To maximize mesh throughput and respect duty cycle regulations 7, we must implement Semantic Compression:
* Dictionary-Based Compression (Smaz/Shoco): For short text strings (e.g., "On my way", "Safe", "Status?"), general-purpose compression algorithms like gzip or zlib often expand the data due to header overhead. We will integrate Shoco or Smaz 8, libraries designed specifically for short string compression. These utilize static frequency models of the English language (or a custom dictionary trained on the "Geometric Language") to compress a 10-byte message into 4-5 bytes.
* Bit-Packing Telemetry: Telemetry data must be ruthlessly bit-packed. A standard JSON object {"voltage":10, "spoons":5} consumes ~25 bytes. By bit-packing, we reduce this to 2 bytes:
   * Voltage (1-10): Requires 4 bits.
   * Spoon Cost (1-5): Requires 3 bits.
   * Status (Enum): Requires 3 bits.
   * Total: 10 bits. Packed into a uint16_t, this leaves 6 bits for flags (e.g., "Emergency", "Silent Mode").
