3.3 The Shortcut Architecture
A major architectural challenge in Google Shared Drives is the rigidity of permission inheritance; by design, one cannot reduce permissions on a child folder or file compared to the parent Shared Drive. If a user is a "Manager" of the Shared Drive, they are a "Manager" of every file within it. This creates a security risk for granular control.
The Geodesic solution is the "Shortcut Architecture":
1. Restrict Root: Users are not given access to the root of the "Backbone" Shared Drive. The root is accessible only to Admins.
2. Granular Sharing: Instead of sharing the drive, specific sub-folders (e.g., 01_SOPs) are shared with specific Google Groups (e.g., "Design Team").
3. User Workspaces: For users who need access to multiple disparate folders across different drives, a dedicated "Workspace" folder is created for them in "Zone Gamma."
4. Shortcuts: The system places shortcuts to the relevant Backbone folders inside their Workspace.
This maintains the structural integrity of the Backbone (files live in one place) while providing a customized, secure "View" for the user. They cannot traverse the directory tree to see unauthorized data, effectively implementing a "Least Privilege" model within a monolithic drive structure. 3

      * Telemetry (Google Sheets): A "Geodesic Telemetry Log" tracks "XP" (words written) and "Burn-Down" rates. This gamification engages the "Race Car Brain" reward system.
      * Fabrication (Google Docs): The system uses a Modular Architecture. Each chapter is a separate Doc to prevent the "Big File Freeze." A "Master Manifest" document contains Smart Chips linking to all chapters, serving as the assembly frame.
      * Pageless Mode: Docs are set to "Pageless" to create an "Infinite Canvas," removing the cognitive friction of page breaks and allowing for large-scale wiring diagrams and tables. 2
