The Geometry of Universal Design

Information Theory
I(X;Y) = H(X) - H(X|Y)
Mutual information. The amount of information shared between source (content) and receiver (user). Accessible design maximizes I(X;Y) for all users, regardless of sensory modality or cognitive processing style.
Shannon Entropy
H(X) = -Σ p(x) log₂ p(x)
Information entropy. Uncertainty in a message. Accessible design reduces entropy by providing multiple channels (visual, auditory, tactile). Redundancy is not waste — it's resilience.

V = 4, E = 6, F = 4
The minimum stable volume. In accessibility terms: four access modes (visual, auditory, motor, cognitive) form the atomic unit. Each mode is a vertex. The edges are the connections between modes. The faces are the interfaces.

Implementation Details

How We Achieve WCAG 2.1 AA

Color Contrast

Phosphorus green (#2ecc71) on near-black (#050510) = 12.6:1 contrast ratio (WCAG AAA). Calcium blue (#60a5fa) on near-black = 8.2:1 (WCAG AA). All text meets or exceeds AA standards.

Text Scaling

All text uses relative units (rem, em, clamp()). Browser zoom works perfectly. No fixed pixel sizes. Text scales from 0.65rem to 5rem without breaking layout.

Keyboard Navigation

Tab order is logical (left-to-right, top-to-bottom). All interactive elements are keyboard accessible. Skip links provided. Focus indicators visible (3px outline, calcium blue).

Screen Reader Support

Semantic HTML throughout. ARIA labels on all interactive elements. Alt text on all images. Descriptive link text. Landmark regions (nav, main, footer).

Animation Control

All animations respect prefers-reduced-motion. CSS media query disables animations when requested. Essential animations only. No auto-playing media.

Form Accessibility

All form inputs have labels. Error messages are descriptive. Required fields are marked. Validation is clear and helpful.

Testing Protocol: Every page is tested with NVDA (Windows), JAWS (Windows), VoiceOver (macOS/iOS), keyboard-only navigation, and automated tools (WAVE, axe DevTools). Validation performed with swarm.sh script on every deployment.

Continuous Improvement: Accessibility is not a one-time check — it's an ongoing process. We welcome feedback. We fix issues within 48 hours. The mesh holds for everyone, or it doesn't hold at all.

Beyond WCAG

Neurodivergent-First Design

WCAG 2.1 AA is the minimum. We go beyond. Neurodivergent-first design means accommodating cognitive differences, sensory sensitivities, and executive function challenges.

Low Cognitive Load

Clear visual hierarchy. Minimal distractions. Focused content. No infinite scroll. No auto-playing media. No pop-ups. Information architecture that respects executive function limits.

Sensory Regulation

High contrast mode. Reduced motion options. No flashing content. Calm color palette. Haptic feedback options (when Node One is connected). Sensory-friendly by default.

Executive Function Support

Clear navigation. Predictable structure. Consistent patterns. No hidden features. No complex interactions. Everything is discoverable, learnable, memorable.

Age-Adaptive

Works for ages 6 to 70+. Simple mode for children. Advanced mode for adults. Adaptive complexity based on user needs. The mesh holds for everyone.

Why Neurodivergent-First: 15–20% of the global population is neurodivergent. Many more have sensory sensitivities or executive function challenges. Designing for neurodivergence benefits everyone. The mesh holds for the most vulnerable, which means it holds for everyone.

The Beauty of Physics: Just as the tetrahedron is the minimum stable volume (accommodating all orientations), neurodivergent-first design is the minimum stable interface (accommodating all users). The geometry itself is inclusive.
