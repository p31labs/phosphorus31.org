Node One

ESP32-S3-based assistive device. Sensory regulation. Off-grid communication. The mesh holds.

Hardware

Specifications

Open-source handheld for sensory regulation and off-grid communication. No internet required. No phone required. No permission required.

ESP32-S3

Dual-core Xtensa LX7 processor. 512KB SRAM. 8MB PSRAM. WiFi + Bluetooth 5.0. USB-C for programming and power.

LoRa 915MHz

SX1262 LoRa transceiver. 915MHz ISM band. Meshtastic firmware. Range: 1-10km line-of-sight. Mesh networking protocol.

DRV2605L Haptic

Linear resonant actuator (LRA) driver. "The Thick Click" haptic feedback. 8-bit waveform library. I2C interface.

OLED Display

128x64 monochrome OLED. SSD1306 controller. I2C interface. Low power consumption. High contrast.

Battery

18650 Li-ion cell. 3.7V nominal. 2500mAh capacity. USB-C charging. Power management IC.

Shell

3D-printed enclosure. PLA/PETG material. Tactile buttons. Haptic feedback zone. Weather-resistant design.

Firmware Deep Dive

ESP-IDF Architecture

FreeRTOS-based firmware. Multi-tasking. Event-driven. The beauty of real-time systems made manifest.

Task Structure

Main Task: System initialization, watchdog, power management. Priority: 1. Stack: 8KB.

LoRa Task: Meshtastic protocol, message routing, mesh networking. Priority: 3. Stack: 12KB.

Haptic Task: DRV2605L driver, waveform playback, pattern library. Priority: 2. Stack: 4KB.

Display Task: SSD1306 driver, menu system, status updates. Priority: 2. Stack: 6KB.

Inter-Task Communication

Queues: FreeRTOS queues for message passing. LoRa → Main → Display. Haptic → Main. Event-driven architecture.

Semaphores: I2C bus semaphore. SPI bus semaphore. Prevents resource conflicts. Ensures thread safety.

Event Groups: System state synchronization. Battery low event. Charging event. Mesh connected event.

Power Management

Deep Sleep: CPU halted. Peripherals powered down. Only RTC active. Current: 10µA. Wake on button press or timer.

Light Sleep: CPU halted. Peripherals active. Current: 1mA. Wake on interrupt. Used for short idle periods.

Active Mode: Full operation. Current: 80–150mA depending on LoRa transmission. Optimized for efficiency.

Mesh Networking

Protocol: Meshtastic (open-source LoRa mesh). Automatic routing. Self-healing. No infrastructure required.

Range: 1–10km line-of-sight. Mesh extends effective range. Each node is a repeater. The geometry amplifies capability.

Bandwidth: 0.350 kbps raw. ~200 bps effective with coding. Sufficient for text, status, sensor data. Not for video or large files.

Why FreeRTOS? Real-time operating system. Deterministic scheduling. Preemptive multitasking. Low overhead. Perfect for embedded systems. The beauty of doing more with less.

Why Event-Driven? Reduces power consumption. Responds to events (button press, incoming message, timer) rather than polling. The system sleeps until needed. Efficiency through physics, not optimization.

Haptic Patterns

"The Thick Click" Library

Sensory regulation through haptic feedback. Not vibration — precise, controlled, therapeutic patterns.

Calm Pattern

Slow, rhythmic pulses. 0.5Hz frequency. 50% intensity. Duration: 30 seconds. Use case: Anxiety reduction, sensory overload recovery.

Alert Pattern

Sharp, brief pulses. 5Hz frequency. 80% intensity. Duration: 1 second. Use case: Important notification, attention redirection.

The Thick Click

Signature pattern. 175Hz fundamental with harmonics. 70% intensity. Duration: 100ms. Use case: General feedback, button confirmation, sensory regulation.

Heartbeat Pattern

Simulated heartbeat. 1.2Hz frequency (72 BPM). 60% intensity. Duration: configurable. Use case: Coherence training, HRV synchronization.

Why Haptics? Sensory regulation is a core need for many neurodivergent individuals. Haptic feedback provides proprioceptive input without requiring visual or auditory attention. It's always available. It's always private. It's always effective.

The Physics: Linear resonant actuators (LRA) operate at resonant frequency (typically 175Hz). This frequency is optimal for sensory regulation — not too high (jarring), not too low (inaudible). The DRV2605L driver provides precise control over waveform, intensity, and duration.
